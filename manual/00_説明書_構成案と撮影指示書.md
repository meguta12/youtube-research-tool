# YouTubeリサーチツール 説明書 構成案 兼 撮影指示書

このドキュメントは「最終的に購入者にお渡しする説明書」を作るための**設計図**です。
- 各セクションで何を書くか
- どの画面のスクショを撮るか
- スクショに何の注釈（矢印・吹き出し）を入れるか
- どこをモザイク・伏字にするか

を一覧化しています。これに沿って撮影 → 加工 → 文章化と進めます。

---

## 想定する読者

- **YouTubeで発信している／これから発信するクリエイター**
- **PCの基本操作（クリック・コピペ・タブ切替）はできる**
- **プログラミング・ITの専門知識は無い**
- ツールを「買って初めて触る」状態

→ **専門用語は出るたびに優しく言い換える。1ステップ1スクショで進める**。

---

## ファイナル形式

- **PDF形式**（マニュアルとしての見栄えと配布のしやすさ）
- A4縦、目次あり、各章先頭にイラスト or アイコン
- スクショは全て**赤い枠＋番号付きの注釈**で「ここを見る／押す」を明示
- 機微情報は**全てモザイクまたは黒塗り**

中間素材として、Markdown版（このリポジトリ内）+ 画像（PNG）を残します。
PDF化は最後に一括で。

## 必ずモザイク・伏字にするもの（撮影時のチェックリスト）

| 対象 | 処理 | 例 |
|---|---|---|
| YouTube APIキー（`AIza...`から始まる39文字） | **黒塗り or `AIza••••••••••••••••••••••••••••••••••` 表示** | 設定画面、入力欄、エラーメッセージ |
| ライセンスキー（`XXXX-XXXX-XXXX-XXXX`） | **黒塗り** | ライセンス入力画面 |
| ユーザーのGoogleアカウントのメール | **黒塗り** | Google Cloud Console画面 |
| ユーザーのGoogleアカウントのプロフィール画像 | **モザイク** | Google Cloud Console右上 |
| Google Cloud プロジェクトID（一意の番号） | **黒塗り** | Google Cloud Console画面 |
| 自分のチャンネル名・自分のメールアドレス | **黒塗り** | 検索結果に映り込んだ場合 |
| 個人を特定できる第三者の情報 | **モザイク** | サムネに映り込んだ顔など |
| ブラウザのブックマークバー | **可能なら撮影前に非表示**（`Cmd+Shift+B`） | スクショ全般 |

→ 撮影フェーズに入ったら、Claude側で機械的に該当箇所を検出してマスクをかけます。

### 第3章：YouTube APIキーの取得（5〜6ページ・最重要）

これは**Google Cloud Console**側の操作なので、Chromeで実際にGoogle Cloud Consoleを開いて撮影します。
非エンジニアの方が一番つまずく箇所なので、**ステップを細かく分けて1操作1スクショ**にします。

#### 3-1. Googleアカウントにログインする
- **撮影対象**：https://console.cloud.google.com/ ログイン直後
- **マスク**：右上のアカウントアイコン、メールアドレス

#### 3-2. プロジェクトを新規作成
- **撮影対象**：上部「プロジェクトを選択」を押した状態 → 「新しいプロジェクト」
- **注釈**：「①ここをクリック」→「②『新しいプロジェクト』を選ぶ」

#### 3-3. プロジェクト名を入力して作成
- **撮影対象**：プロジェクト名入力フォーム
- **注釈**：「①任意の名前（例：YouTubeリサーチ用）」「②『作成』を押す」
- **マスク**：プロジェクトID（自動生成される番号）

#### 3-4. YouTube Data API v3 を有効化
- **撮影対象**：「APIライブラリ」→「YouTube Data API v3」検索結果
- **撮影対象2**：API詳細ページの「有効にする」ボタン

#### 3-5. APIキーを発行
- **撮影対象**：「認証情報」画面の「+ 認証情報を作成」メニュー
- **撮影対象2**：「APIキー」を選んだ直後の「APIキーを作成しました」ダイアログ
- **マスク**：**ダイアログ内のAPIキー本体（黒塗り）**

#### 3-6. APIキーをコピー
- **撮影対象**：コピーアイコンを押す前後
- **マスク**：APIキー本体

#### 3-7. ⚠️ APIキーに制限をかける（漏洩対策）
- **撮影対象**：作成したキーの「鉛筆」アイコンを押した詳細画面
- **撮影対象2**：「アプリケーションの制限」で「ウェブサイト」を選択
- **撮影対象3**：「ウェブサイトの制限事項」にツールURLを `/*` 付きで追加
- **撮影対象4**：「APIの制限」で「キーを制限」→ YouTube Data API v3 にチェック
- **撮影対象5**：「保存」を押した状態
- **マスク**：APIキー本体

→ **このセクションが説明書の心臓部**。図解とチェックリスト形式を併用。

### 第14章：困ったときは（FAQ・トラブルシューティング）（2ページ）

スクショ不要、文章のみ。以下を網羅：

| 症状 | 原因 | 対処 |
|---|---|---|
| 「YouTube APIエラー: API key not valid」 | キーの貼り付けミス、または制限が強すぎ | 設定画面で再入力／Cloud Consoleで制限を確認 |
| 「YouTube APIエラー: quotaExceeded」 | 1日10,000ユニットを使い切った | 翌日（日本時間17時）に自動リセット |
| 「該当する動画が見つかりませんでした」 | キーワードが具体的すぎ／期間が短すぎ | 一般的な語に／期間を広げる |
| 大手公式ばかり並ぶ | フィルタなし | 設定で「除外チャンネルID」を追加 |
| 結果が出るのが遅い | 取得件数が多い | まずは25件で試す |
| ライセンスキーが通らない | 入力ミス／違うキーを入れている | 購入時メールを再確認 |
| 履歴・APIキーが消えた | ブラウザのCookie/履歴削除 | 設定し直し（バックアップはExcel） |

## 撮影フェーズの進め方

1. **準備**
   - Chromeで「シークレットウィンドウ」を開く（ブックマーク・履歴非表示のため）
   - ウィンドウ幅を**1280px**に固定（説明書のレイアウト統一）
   - ブラウザのズーム100%

2. **ダミーデータの用意**
   - 仮ライセンスキー：`DEMO-1234-5678-DEMO`
   - 仮APIキー：実物を使うが、撮影後に必ずマスク処理
   - 検索キーワード：個人特定につながらない一般語（例：「副業」「ダイエット」「英語学習」）

3. **撮影順序**（依存関係に注意）
   1. ライセンス画面 → 入力 → オンボーディング1〜3 → ホーム
   2. Google Cloud Console（ブラウザ別タブで）
   3. 検索実行 → 各タブを巡回
   4. 設定モーダル、ヘルプモーダル
   5. Excel エクスポート結果

4. **マスク処理**
   - スクショごとに「マスク必要箇所リスト」を併記
   - 機械的にマスク（黒塗りまたはガウシアンブラー）
   - 完了後、人間（あなた）が最終目視チェック

5. **保管**
   - `manual/screenshots/raw/` … 元画像
   - `manual/screenshots/masked/` … マスク済み（こちらを本番採用）
   - `manual/screenshots/annotated/` … 注釈入り（最終）
